# Phase 4.2: `_graphql` Parameter on FHIR Operations

**Date:** 2026-05-02
**Status:** Proposed
**Scope:** Close the `_graphql` query parameter gap for FHIR operations
**Reference:** [FHIR GraphQL Specification — _graphql parameter](https://build.fhir.org/graphql.html)

---

## Problem Statement

The FHIR GraphQL specification allows applying a GraphQL query to the result of any FHIR RESTful operation via the `_graphql` query parameter:

```
GET [base]/ValueSet/x/$validate-code?_graphql={result:parameter(name:"result"){value:valueBoolean}}
```

The middleware (`GraphQlOperationMiddleware`) already captures operation responses when `_graphql` is present, but currently executes a standard GraphQL query against the schema root — it has no way to bind the operation's JSON result as the data source.

This was previously deferred because HotChocolate v15 lacks `SetRootValue()` on `OperationRequestBuilder`. However, we can solve this using the same wrap/unwrap pattern already proven for instance-level queries.

---

## Approach: Wrap/Unwrap with Nullable `id` + Global State

### How Instance Queries Already Solve This

Instance-level GraphQL queries (`/Patient/123/$graphql`) face the same problem: the user's selection set applies directly to a resource, but the schema root expects `Query.Patient(id:)`. The solution:

1. **Wrap:** `{ id name }` -> `{ Patient(id: "123") { id name } }`
2. **Execute:** Standard GraphQL against the schema root
3. **Unwrap:** Extract `Patient` from result, hoist fields to root

### Applying to Operation Results

Operation results are FHIR resources (e.g., `Parameters`, `Bundle`, `OperationOutcome`). We use the identical pattern:

1. **Parse** the operation response as `JsonElement`
2. **Extract** `resourceType` (e.g., `"Parameters"`)
3. **Wrap** the user's query inside `{ <resourceType> { ... } }`
4. **Set global state** `OperationResult` with the parsed `JsonElement`
5. **Execute** via `IGraphQlExecutionService`
6. **Unwrap** the result by extracting the `<resourceType>` field value

The single-resource read field (`Patient`, `Parameters`, etc.) checks global state when no `id` argument is provided, returning the operation result directly.

---

## Why This Works

- **No HC internals needed:** Uses only public APIs (`AddGlobalState`, `OperationRequestBuilder`)
- **Schema bloat avoided:** No new root fields or special types
- **Proven pattern:** Instance queries use the exact same wrap/unwrap logic
- **Type-safe:** The resource type from the JSON response determines the wrapper field

## Detailed Design

### 1. IGraphQlExecutionService Extension

Add an overload that accepts a dictionary of global state entries:

```csharp
Task<IExecutionResult> ExecuteAsync(
    GraphQlRequestBody request,
    FhirVersion version,
    IReadOnlyDictionary<string, object?> globalState,
    CancellationToken cancellationToken);
```

The existing `ExecuteAsync` and `ExecuteInstanceAsync` delegate to a shared core that accepts optional global state.

### 2. ResourceResolver Changes

Change `ResolveByIdAsync` signature to accept nullable `id`:

```csharp
public async Task<JsonElement?> ResolveByIdAsync(
    string resourceType,
    string? id,
    IResolverContext context,
    CancellationToken cancellationToken)
```

If `id` is null:
1. Read `OperationResult` from global state
2. Verify `resourceType` matches the expected type
3. Return the operation result JsonElement

If `id` is null and no `OperationResult` exists, throw a GraphQL error: `"id is required unless querying via _graphql parameter"`.

### 3. FhirTypeModule Changes

In `BuildQueryType`, change the single-resource read field from:

```csharp
descriptor.Field(capturedType)
    .Argument("id", a => a.Type<NonNullType<IdType>>())
```

to:

```csharp
descriptor.Field(capturedType)
    .Argument("id", a => a.Type<IdType>())
```

Pass `context` to the resolver so it can access global state:

```csharp
.Resolve(async ctx =>
{
    var id = ctx.ArgumentValue<string?>("id");
    var resolver = ctx.Service<AppResourceResolver>();
    return await resolver.ResolveByIdAsync(capturedType, id, ctx, ctx.RequestAborted);
});
```

### 4. GraphQlOperationMiddleware Rewrite

Replace the TODO-limited implementation with full wrap/unwrap logic:

```csharp
public async Task InvokeAsync(HttpContext context)
{
    var graphQlQuery = context.Request.Query["_graphql"].FirstOrDefault();
    if (string.IsNullOrEmpty(graphQlQuery))
    {
        await next(context);
        return;
    }

    var originalBody = context.Response.Body;
    using var capturedBody = new MemoryStream();
    context.Response.Body = capturedBody;

    try
    {
        await next(context);

        if (context.Response.StatusCode >= 400)
        {
            capturedBody.Seek(0, SeekOrigin.Begin);
            await capturedBody.CopyToAsync(originalBody);
            return;
        }

        capturedBody.Seek(0, SeekOrigin.Begin);
        JsonElement operationResult;
        try
        {
            operationResult = await JsonSerializer.DeserializeAsync<JsonElement>(capturedBody, cancellationToken: context.RequestAborted);
        }
        catch (JsonException)
        {
            capturedBody.Seek(0, SeekOrigin.Begin);
            await capturedBody.CopyToAsync(originalBody);
            return;
        }

        if (!operationResult.TryGetProperty("resourceType", out var rtProp)
            || rtProp.GetString() is not string resourceType)
        {
            // Not a FHIR resource — pass through
            capturedBody.Seek(0, SeekOrigin.Begin);
            await capturedBody.CopyToAsync(originalBody);
            return;
        }

        var executionService = context.RequestServices.GetRequiredService<IGraphQlExecutionService>();
        var contextAccessor = context.RequestServices.GetRequiredService<IFhirRequestContextAccessor>();
        var version = contextAccessor.RequestContext?.FhirVersion ?? FhirVersion.R4;

        // Wrap the user's query inside the resource type field
        var wrappedQuery = WrapOperationQuery(graphQlQuery, resourceType);

        var requestBody = new GraphQlRequestBody(wrappedQuery, null, null);
        var globalState = new Dictionary<string, object?>
        {
            ["OperationResult"] = operationResult,
        };

        var result = await executionService.ExecuteAsync(requestBody, version, globalState, context.RequestAborted);

        // Unwrap: extract the resource type field from data
        result = UnwrapOperationResult(result, resourceType);

        context.Response.ContentType = "application/json; charset=utf-8";
        context.Response.StatusCode = StatusCodes.Status200OK;
        await ResultFormatter.FormatAsync(result, originalBody, context.RequestAborted);
    }
    finally
    {
        context.Response.Body = originalBody;
    }
}

private static string WrapOperationQuery(string query, string resourceType)
{
    var trimmed = query.Trim();
    if (trimmed.StartsWith('{') && trimmed.EndsWith('}'))
    {
        var inner = trimmed[1..^1].Trim();
        return $"{{ {resourceType} {{ {inner} }} }}";
    }
    return query;
}

private static IExecutionResult UnwrapOperationResult(IExecutionResult result, string resourceType)
{
    if (result is not IOperationResult { Data: { } data } opResult)
        return result;

    if (!data.TryGetValue(resourceType, out var resourceData))
        return result;

    if (resourceData is not IReadOnlyDictionary<string, object?> resourceDict)
        return result;

    return OperationResultBuilder
        .FromResult(opResult)
        .SetData(resourceDict)
        .Build();
}
```

### 5. GraphQlExecutionService Changes

Extract `WrapInstanceQuery` and `UnwrapInstanceResult` as reusable helpers. Add the global state overload:

```csharp
public Task<IExecutionResult> ExecuteAsync(
    GraphQlRequestBody request,
    FhirVersion version,
    CancellationToken cancellationToken)
    => ExecuteCoreAsync(request, version, null, null, null, cancellationToken);

public Task<IExecutionResult> ExecuteInstanceAsync(
    GraphQlRequestBody request,
    FhirVersion version,
```

## E2E Test Plan

### Test: `_graphql` on `$validate-code`

**Setup:**
1. Create a ValueSet resource
2. Call `$validate-code` with `_graphql` parameter

**Query:**
```
GET /ValueSet/{id}/$validate-code?code=test&system=http://example.org&_graphql={result:parameter(name:"result"){value:valueBoolean}}
```

**Assertions:**
- Response status is 200
- Response is valid GraphQL JSON
- `data.result.value` is a boolean (true/false depending on validation)

### Test: `_graphql` on operation returning OperationOutcome

**Setup:**
1. Call an operation that returns an error OperationOutcome
2. Include `_graphql` parameter

**Assertions:**
- The operation's error response is returned as-is (no GraphQL transformation)
- Status code preserved

## Edge Cases

| Case | Behavior |
|------|----------|
| Operation returns non-JSON | Pass through unchanged |
| Operation returns JSON without `resourceType` | Pass through unchanged |
| GraphQL query has syntax errors | Return GraphQL error response (standard HC behavior) |
| `id` is null and no `OperationResult` in global state | Return GraphQL error: `"id is required unless querying via _graphql parameter"` |
| `_graphql` on failed operation (>=400) | Pass through error response unchanged |
